## 一、产品背景

### 1.1 市场机会

中小企业和独立运营者在多渠道营销中面临共同的痛点：
- 撰写不同渠道（官网、朋友圈、小红书、抖音、邮件）的营销文案耗时费力
- 保持品牌调性一致的同时适配各平台风格困难
- 优质文案外包成本高，内部产出效率低

### 1.2 产品定位

**CopyAI** 是一款面向中文市场的 AI 营销文案生成 SaaS 平台。用户输入产品信息，AI 自动生成适配多投放渠道的营销文案，支持官网、朋友圈、小红书、抖音、邮件等场景。

### 1.3 目标用户

| 用户画像 | 特征 | 需求 |
|----------|------|------|
| 独立创业者 | 单人/小团队，预算有限 | 快速生成多版本文案，免费可用 |
| 运营人员 | 负责社交媒体/电商运营 | 多渠道内容产出，保持品牌一致 |
| 内容工作室 | 服务多个客户，团队协作 | 批量生成，优先生成队列 |

## 二、功能范围

### 2.1 核心功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| AI 文案生成 | 输入产品名、描述、目标用户、卖点、渠道，AI 自动生成标题/副标题/CTA/短文案/长文案 | P0 |
| 多渠道适配 | 官网、朋友圈、小红书、抖音、邮件五种渠道的差异化文案 | P0 |
| 结果复制/导出 | 一键复制各文案片段 | P1 |
| 生成历史管理 | 历史记录查看、筛选、复用 | P1 |
| 用户认证 | 邮箱+密码注册/登录，邮箱确认 | P0 |
| 套餐管理 | Free / Pro 两级套餐 | P0 |
| Stripe 支付 | Checkout 订阅 + Customer Portal 管理 | P0 |
| 使用限制 | Free 用户每日 3 次生成 | P0 |
| 管理后台 | 用户列表、生成记录、订阅状态查看 | P1 |

### 2.2 不在本次范围内

- 团队协作与多成员工作区
- AI 模型训练/微调（使用 DeepSeek 通用模型）
- 社交媒体直接发布
- 多语言支持（当前仅中文）

## 三、用户流程

### 3.1 核心流程

```
营销官网 (www) → 注册 → 邮箱确认 → 登录
  → 工作台（/generate）→ 填写产品信息 → AI 生成 → 查看/复制结果
  → 历史记录（/history）→ 筛选/复用
  → 套餐管理（/billing）→ 选择 Pro → Stripe 支付 → 自动升级
```

### 3.2 认证流程

```
注册 → 填写邮箱密码 → Supabase 发送确认邮件
  → 用户点击确认 → /auth/callback 处理 → 登录成功
  → 跳转到 /generate
```

### 3.3 支付流程

```
用户选择 Pro 套餐 → 月付/年付选择 → POST /api/stripe/checkout
  → Stripe Checkout 页面 → 用户支付
  → 成功回调 → /billing?success=true
  → Webhook（checkout.session.completed）→ 更新 profile 套餐 → 订阅激活
```

### 4.2 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| 框架 | Next.js 14 (App Router) | 全栈 Web 框架 |
| 语言 | TypeScript | 类型安全 |
| 样式 | Tailwind CSS 3 | 原子化 CSS |
| Monorepo | pnpm 9 + Turborepo | 多包管理 |
| 数据库 | Supabase (PostgreSQL) | 用户数据、生成记录 |
| 认证 | Supabase Auth | 邮箱密码 + RLS |
| 支付 | Stripe Checkout / 订阅 | 付费套餐 |
| AI | DeepSeek API (deepseek-chat) | 文案生成 |
| 部署 | Vercel (3 个项目) | 云托管 |

## 六、非功能需求

### 6.1 安全

- Cookie-based session（httpOnly, SameSite）
- Supabase RLS 行级安全策略
- Stripe Webhook 签名验证
- 服务端 API key 不暴露给前端
- 速率限制：Free 用户 3 次/日

### 6.2 性能

- AI 生成请求 ≤ 10s 超时
- 页面加载 ≤ 2s (LCP)
- 静态页面 ISR 增量静态再生

### 6.3 可维护性

- Monorepo 结构，共享类型与组件
- Turborepo 增量构建缓存
- 完整的数据库迁移脚本
